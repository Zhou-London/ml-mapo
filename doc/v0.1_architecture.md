# ML-MAPO Prototype — Architecture

## Topology

```
          Data Module
                |                           
                V                           
Forecast Module & Risk Module
                |
                |
                V
       Optimization Module
```

Each module is a standalone Python program, sharing data through ZeroMQ PUB/SUB.
The runner (`prototype/main.py`) launches all four as subprocesses, starting
subscribers first so SUB sockets are connected before `data` publishes.

## Modules

| Module         | In                     | Out                | Job                         |
| -------------- | ---------------------- | ------------------ | --------------------------- |
| `data`         | yfinance / TimescaleDB | PUB 5555 (`OHLCV`) | cache + broadcast snapshots |
| `risk`         | SUB 5555               | PUB 5556 (`COV`)   | covariance matrix           |
| `forecast`     | SUB 5555               | PUB 5557 (`ALPHA`) | combined forecasting vector |
| `optimization` | SUB 5556, SUB 5557     | stdout             | MVO weights                 |

`risk` and `forecast` are parallel — they never talk to each other.

## Wire format

ZMQ multipart: `[topic_bytes, pickle.dumps(payload)]`. Payload is a `dict`
carrying pandas DataFrames / Series. Pickle ⇒ all modules must share a
Python version.

| Topic   | Payload keys                                            |
| ------- | ------------------------------------------------------- |
| `OHLCV` | `tickers`, `start`, `end`, `ohlcv: {ticker: DataFrame}` |
| `COV`   | `factor`, `tickers`, `covariance: DataFrame`            |
| `ALPHA` | `factors`, `tickers`, `alpha: Series`                   |

## Data module

- ORM: SQLAlchemy 2.0 `OHLCV(ticker, ts, open, high, low, close, adj_close, volume)`
  with composite PK `(ticker, ts)`. Promoted to a TimescaleDB hypertable on `ts`.
- DB URL hard-coded: `postgresql+psycopg2://postgres:password@localhost:6543/postgres`.
- Incremental fetch: `find_missing_ranges` returns at most a leading and a
  trailing gap inside `[start, end]`. Interior holes (weekends/holidays)
  are ignored. Upsert via `session.merge`.
- Publishes the snapshot every `PUBLISH_INTERVAL_S` (5s) so late
  subscribers still receive data.

### Adaptor interface

`prototype/data/adaptors.py`:

```python
class DataSourceAdaptor(ABC):
    name: str
    @abstractmethod
    def fetch_ohlcv(self, ticker, start, end) -> pd.DataFrame: ...
```

Contract: `[start, end]` inclusive, return a DataFrame indexed by date with
columns `OHLCV_COLUMNS = ["Open","High","Low","Close","Adj Close","Volume"]`,
or an empty frame. Implementations: `YfAdaptor`. Add new sources by
subclassing and swapping the instance constructed in `data/main.py:main`.

## Risk module

```python
class RiskFactor(ABC):
    name: str
    @abstractmethod
    def covariance(self, ohlcv) -> pd.DataFrame: ...
```

Default: `NaiveRiskFactor` — annualized sample covariance of daily log returns
on `adj_close`, lookback 252.

## Forecast module

```python
class AlphaFactor(ABC):
    name: str
    @abstractmethod
    def score(self, ohlcv) -> pd.Series: ...
```

Default: `NaiveMomentumAlpha` — 12-1 momentum (lookback 252, skip 21),
annualized. Multiple factors are combined by `ir_weighted_combine`:
z-score each factor, weight by IR, rescale to a return magnitude.

## Optimization module

`mean_variance_optimize(alpha, cov, λ, long_only)` — SLSQP solves

```
max  α'w − (λ/2) w'Σw
s.t. 1'w = 1,  w ≥ 0  (if long_only)
```

with an analytic gradient. Defaults: `λ = 50`, long-only. Lower `λ` ⇒
corner solutions; higher ⇒ more diversified.

## Lifecycle

- Each child installs `signal.signal(SIGTERM, signal.default_int_handler)` so
  SIGTERM raises `KeyboardInterrupt` and the `finally` block closes sockets
  with `linger=0` and `ctx.term()`.
- Runner spawns children with `start_new_session=True`. Terminal Ctrl+C
  signals only the runner; the runner forwards a clean SIGTERM. Stragglers
  past `SHUTDOWN_TIMEOUT_S = 10` are SIGKILLed.

## Run

```
uv sync
uv run python prototype/main.py     # Ctrl+C to stop
```

Logs go to stdout. To capture to a file, set `PYTHONUNBUFFERED=1`,
otherwise child output is block-buffered when stdout is not a TTY:

```
PYTHONUNBUFFERED=1 uv run python -u prototype/main.py 2>&1 | tee run.log
```

Each module is also runnable standalone (`uv run python prototype/<mod>/main.py`),
but downstream modules block on receive until upstream is publishing.
